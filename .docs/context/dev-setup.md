🛠️ Workspace Configuration Guide
This document outlines the optimizations in .vscode/settings.json designed to maximize IDE performance and AI context precision.

🦀 Rust-Analyzer (RA) Optimizations
The rust-analyzer engine is the core of the Rust development experience. In this monorepo structure, explicit configuration is required for the IDE to resolve dependencies correctly.

rust-analyzer.linkedProjects
Action: Explicitly maps the location of Cargo.toml files (./api and ./app).

For the Agent: Use these paths as project roots. Without this, global indexing and cross-crate "Go to Definition" will fail.

rust-analyzer.cargo.buildScripts.enable
Action: Allows RA to execute build.rs scripts.

For the Agent: Necessary for detecting dynamically generated code. This prevents "false positive" errors in the editor and ensures the AI sees the complete API surface.

rust-analyzer.procMacro.enable
Action: Enables expansion of procedural macros (e.g., #[derive(Serialize)]).

For the Agent: Allows the AI to "read" the code generated by macros, ensuring that autocomplete and code suggestions are 100% accurate.

⚡ Search & AI Performance
AI tools scan the workspace to build a context window. Including redundant data slows down inference and degrades the quality of suggestions.

search.exclude
We have excluded the following directories from indexing:

**/target: Rust compilation output.

**/artifact_pkg: Build artifacts and distribution packages.

Why this matters for AI Agents:

Token Efficiency: Prevents the agent from wasting the context window on unreadable binary data or build logs.

Accuracy: Stops the AI from referencing outdated build artifacts instead of the current source code.

Performance: Dramatically reduces CPU/RAM load during workspace indexing.

🧩 Extensions & Templates
Extensions: If a specific tool is required for the workflow, suggest adding it to .vscode/extensions.json so the AI and new contributors receive automatic setup prompts.

Local Configuration:

Active Settings: See .vscode/settings.json.

Template: Use .vscode/settings.json.example to customize your environment without affecting the shared repository.

Note for AI Agents: Always ignore directories listed in search.exclude. Focus analysis strictly on the source code within the directories specified in linkedProjects.
